# Module 1.2: Visualizing Files

## Reference Guide

- **Time to Complete:** 10-15 minutes
- **Prerequisites:** Module 1.1 complete; course folder open in your terminal

> **Start this lesson interactively:** Type `start-1-2` in Codex to set up your workspace step-by-step.

## Overview

Codex runs in your terminal, which is powerful but can feel like flying blind if you can't *see* your files. This lesson sets up a **visual workspace** — a side-by-side layout where you watch files appear and change in real time as Codex works.

**Key takeaway:** Pair your terminal (where you talk to Codex) with a file viewer (where you see the results). You'll never wonder "what just happened?" again.

## Why Visualization Matters

With a visual workspace open, you can:

- See your entire project structure at a glance
- Watch PRDs, research notes, and summaries appear as Codex creates them
- Edit files yourself when you want to
- Review documents without switching apps

## Choose a File Viewer

You have two main options. Either works — pick what fits how you already work.

| Tool | Best for | Notes |
|------|----------|-------|
| **Obsidian** | PMs working mostly with Markdown docs | Free, beginner-friendly, great for notes. Cannot show hidden folders (anything starting with a dot) — use Finder/Explorer for those. Download at [obsidian.md](https://obsidian.md/) |
| **VS Code** | Anyone who wants more power, or already codes | Shows hidden folders. Press `Cmd+Shift+.` (Mac) to reveal them. |

### Opening your course folder

In either tool, open your course folder as the workspace (Obsidian calls this a "vault"; VS Code uses File → Open Folder). If you're not sure where the folder lives, ask Codex — it can print the full path for you. You should then see folders like `company-context/` and your lesson files in the sidebar.

## Set Up Split-Screen

The most common way to work with Codex:

- **Left half of your screen:** the terminal running Codex
- **Right half of your screen:** your file viewer (Obsidian or VS Code)

Resize and position the two windows side by side. Now you can chat with Codex on the left and watch the files it creates and edits on the right, all at once.

Other layouts work too — terminal full-screen with quick app-switching, a three-way split with a browser, or dual monitors. The only requirement is that you can see your files while Codex works.

## The Workflow From Here On

This is the loop you'll use for the rest of the course:

1. You ask Codex to do something (summarize notes, draft a PRD, analyze research).
2. Codex does it.
3. You **see** it happen in your file viewer.
4. You review, edit, or ask for revisions.
5. Repeat.

## Quick Tips

- Click files in the sidebar to open them; most viewers auto-refresh when a file changes.
- **Obsidian:** `Cmd/Ctrl + O` to quick-search files; `Cmd/Ctrl + E` toggles edit/reading view. Remember hidden folders won't show — use Finder/Explorer when you need them.
- **VS Code:** `Cmd/Ctrl + P` to quick-open files; the explorer shows everything, including hidden folders.

You don't need to master your editor right now. Just use it as a file viewer — you'll pick up features naturally as you go.
